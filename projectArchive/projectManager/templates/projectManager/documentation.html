Web Team Project Archive Documentation

Overview

The Web Team Project Archive is a Django-based application designed to archive and manage details about web-related projects. The app supports the submission of project information, contributors, approvers, and related webpage links. It also includes tools to view, filter, and sort all submitted projects.

Project Submission Workflow

  1. Users navigate to the Add Project Form.

  2. Users fill out the Add Project Form, entering core project details.

  3. Dynamic forms allow adding team members, approvers, and webpage URLs.

  4. If a person or webpage already exists, the existing record is reused instead of creating a new one.

  5. The form validates all data, including email uniqueness and URL duplication, before saving.

Project Model Structure

Each project includes the following fields:

  • Name: Project title
  • Start/End Dates: Duration of the project
  • Description: Optional text describing the project
  • Team Members: Linked via a many-to-many relationship to Person
  • Approvers: Also linked to Person via a separate many-to-many field
  • Webpages: URLs related to the project, also many-to-many
  • Email/Comparison PDF URLs: Optional links to supplementary materials

Person Model

  • First Name
  • Last Name
  • Email (must be unique)

Each Person can be involved in multiple projects as either a team member or an approver.

Webpage Model

  • URL (must be unique)

Webpages are linked to one or more projects.

Views

  • Home Page: Displays recent projects and contributors
  • All Projects: Table view of all projects, sortable and filterable
  • Project Detail: Full information on a single project
  • Add Project: Form to submit new project information
  • About: Static info about the archive
  • Documentation: This page

Dynamic Forms

The Add Project form supports dynamic addition and removal of:

  • Team members
  • Approvers
  • Webpages

Validation ensures no duplicate emails or URLs are submitted.

Technologies Used

  • Django (views, models, forms, and templates)
  • Bootstrap 5 for UI styling and responsiveness
  • JavaScript for dynamic form handling

Future Enhancements

  • Search by person or webpage
  • Role tagging for team members and approvers
  • Export project data to CSV or PDF
  • Authentication and access control for project editing
